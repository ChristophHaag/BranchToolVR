puntoexe::imebra::directoryRecord Class Reference
Dicomdir

Detailed Description

directoryRecord represents a single record in a DICOMDIR structure (see dicomDir).

A new directoryRecord object cannot be constructed directly but only by calling dicomDir::getNewRecord() on the directory the new record will belong to.

#include <dicomDir.h>

std::wstring puntoexe::imebra::directoryRecord::getFilePart ( std::uint32_t  part)

Get the full path to the file referenced by the record.

Several calls to this function may have to be made to obtain the full file path.

The full file path is formed by a list of folder and by the file name; for each call to this function the caller has to specify a zero based part number.

The last valid part contains the file name, while the parts before the last one contain the folder in which the file is stored; each folder is a child of the folder returned by the previous part.

The function returns an empty string if the specified part doesn't exist.

See also setFilePart().

partthe part to be returned, 0 based.

void puntoexe::imebra::directoryRecord::setFilePart ( std::uint32_t  part,
const std::wstring  partName 
)

Set the full path to the file referenced by the record.

Several calls to this function may have to be made to set the full file path.

The full file path is formed by a list of folder and by the file name; for each call to this function the caller has to specify a zero based part number.

The last valid part contains the file name, while the parts before the last one contain the folder in which the file is stored; each folder is a child of the folder set in the previous part.

For instance, the file /folder/file.dcm is set with two calls to setFilePart():

  • setFilePart(0, "folder")
  • setFilePart(1, "file.dcm")

See also getFilePart().

Parameters
partthe part to be set, 0 based.
partNametha value to set for the part
